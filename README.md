# FF-punch

FF-punch is a neutral security-awareness phishing website demo project for controlled training, interface prototyping, and safe experimentation.

## Overview

This repository is intended to support legitimate education and internal training only. It must not be used to collect credentials, impersonate users, or access accounts without explicit authorization.

## Project Structure

- `run.py` - entry point used for local development experiments.
- `linux.sh` - convenience script for Linux-based environments.
- `termux.sh` - convenience script for Termux-based environments.
- `templates/` - static assets and example page layouts.

## Safety Notice

- Do not use this project to harvest passwords, tokens, or account data.
- Do not deploy it against real users or services.
- Keep all testing confined to systems you own or are explicitly permitted to use.

## Setup

If you are using this repository for approved training or UI prototyping, review the scripts and templates before running anything locally.

## Development

Use the files in this repository as a starting point for benign demonstrations, mock login flows, or security-awareness exercises that do not process real credentials.
